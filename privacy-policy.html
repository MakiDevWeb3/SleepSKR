Privacy Policy

SleepSkr privacy policy

No ads. No tracking. Sleep data stays on your device. Wallet use is optional, and SleepSkr never receives your private key.

01

Stored locally

Sleep sessions, settings, reminders, and notes are saved on the device.

02

Not collected

No ads, no analytics, and no personal profile data are collected by the app.

03

Wallet optional

Users can stay in guest mode and use the app without connecting a wallet.

Permissions

Android permissions explained in plain English

  • android.permission.ACCESS_NETWORK_STATE Not requested

    Lets an app check whether the device has a network connection. SleepSkr does not need this permission in the current build.

  • android.permission.INTERNET Used

    Allows user-started network requests such as wallet or donation calls. It is not used for ads or tracking.

  • android.permission.RECEIVE_BOOT_COMPLETED Used

    Lets the app reschedule reminders after a phone restart.

  • android.permission.SCHEDULE_EXACT_ALARM Not requested

    The current code uses WorkManager for reminders, so exact alarms are not needed in this release.

  • com.taqueur.sommeil.DYNAMIC_RECEIVER_NOT_EXPORTED_PERMISSION Library

    Internal AndroidX permission used to keep dynamically registered receivers private to the app.

  • android.permission.FOREGROUND_SERVICE Not requested

    Would be needed only for a visible long-running foreground task. SleepSkr does not run one in the current build.

  • android.permission.POST_NOTIFICATIONS Used

    Allows sleep reminders and wake-up notifications. Users can disable notifications in Android settings.

  • android.permission.REORDER_TASKS Not requested

    Lets an app move tasks in Android's recent-apps list. SleepSkr does not use this feature.

  • android.permission.WAKE_LOCK Used

    Helps reminder work continue reliably while the device is asleep or in doze mode.

Wallet

Optional wallet behavior

Wallet use is optional. If a user connects a wallet, SleepSkr only reads the public address to display it locally. It does not access a private key, seed phrase, or recovery phrase.

Donations are voluntary. Any blockchain donation is a real transaction the user approves in their own wallet. Blockchain transactions are irreversible once broadcast.

Data handling

What is stored and what is not

  • Local first Sleep timing, night duration, and user notes remain on the device.
  • No ads No advertising SDKs or tracking pixels are included.
  • No private keys Private wallet credentials are never collected or stored.
  • Delete any time Clearing app data or uninstalling removes locally stored data.
